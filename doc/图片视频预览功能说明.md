# 图片和视频预览功能添加说明

版本：v0.2  
变更日期：2026-05-09  
涉及范围：后端预览接口、文件类型识别、SQLite 约束迁移、前端管理台预览弹窗

## 1. 功能概述

本次更新在原有 `text`、`markdown`、`pdf` 预览基础上，新增：

1. 图片预览：`preview_kind = "image"`
2. 视频预览：`preview_kind = "video"`

管理员在文件列表中可以筛选图片和视频文件，并通过预览按钮直接查看图片或播放视频。访客下载逻辑不变，仍然只能凭 6 位文件码下载。

## 2. 后端修改详情

### 2.1 `preview_kind` 新增枚举值

原枚举：

```text
none, text, markdown, pdf
```

新枚举：

```text
none, text, markdown, pdf, image, video
```

影响字段：

```text
files.preview_kind
```

影响接口响应：

```text
POST /api/v1/files
GET /api/v1/admin/files
GET /api/v1/admin/files/{file_id}
```

### 2.2 SQLite 迁移

因为 `files.preview_kind` 有 CHECK 约束，本次更新增加了启动时自动迁移：

1. 检测当前 `files` 表定义是否已包含 `image` 和 `video`。
2. 如果是旧约束，则创建 `files_new`。
3. 拷贝原有文件元数据。
4. 替换旧表并重建索引。
5. 写入 `schema_migrations` 版本 `2`。

已有上传文件和事件记录会保留。

### 2.3 MIME 和预览类型判定

后端上传时仍然流式写入文件，同时读取前 512 字节用于 MIME 辅助识别。为了让常见媒体文件稳定预览，本次增加了扩展名兜底。

图片支持：

| 扩展名 | MIME |
| --- | --- |
| `.jpg`、`.jpeg` | `image/jpeg` |
| `.png` | `image/png` |
| `.gif` | `image/gif` |
| `.webp` | `image/webp` |
| `.bmp` | `image/bmp` |
| `.avif` | `image/avif` |

视频支持：

| 扩展名 | MIME |
| --- | --- |
| `.mp4`、`.m4v` | `video/mp4` |
| `.webm` | `video/webm` |
| `.ogv`、`.ogg` | `video/ogg` |
| `.mov` | `video/quicktime` |

说明：

1. SVG 未纳入图片预览，避免把可脚本化内容作为内联预览暴露。
2. 其他浏览器无法播放的视频编码，即使后端返回 `video`，前端播放器也可能无法播放。

### 2.4 预览接口行为

接口：

```http
GET /api/v1/admin/files/{file_id}/preview
Authorization: Bearer <access_token>
```

响应规则：

| `preview_kind` | 响应 |
| --- | --- |
| `text` | JSON，返回文本内容 |
| `markdown` | JSON，返回 Markdown 原文 |
| `pdf` | 内联二进制流 |
| `image` | 内联图片流 |
| `video` | 内联视频流 |
| `none` | `422 UNSUPPORTED_PREVIEW` |

图片响应示例：

```http
HTTP/1.1 200 OK
Content-Type: image/png
Content-Disposition: inline; filename="pixel.png"; filename*=UTF-8''pixel.png
Accept-Ranges: bytes
```

视频响应示例：

```http
HTTP/1.1 200 OK
Content-Type: video/mp4
Content-Disposition: inline; filename="clip.mp4"; filename*=UTF-8''clip.mp4
Accept-Ranges: bytes
```

视频预览通过 `http.ServeContent` 返回，支持浏览器 Range 请求。

### 2.5 管理员文件列表筛选

接口：

```http
GET /api/v1/admin/files?preview_kind=image
GET /api/v1/admin/files?preview_kind=video
```

`preview_kind` 现在支持：

```text
none, text, markdown, pdf, image, video
```

## 3. 前端修改详情

### 3.1 类型更新

`src/types.ts`：

```ts
export type PreviewKind = "none" | "text" | "markdown" | "pdf" | "image" | "video";
```

### 3.2 显示文本更新

`src/utils.ts` 新增：

```ts
image: "图片"
video: "视频"
```

### 3.3 管理端筛选项更新

文件列表的预览类型筛选新增：

```text
图片
视频
```

对应查询参数：

```text
preview_kind=image
preview_kind=video
```

### 3.4 预览弹窗更新

`src/App.tsx` 中，预览逻辑变为：

1. `text`、`markdown`：调用 `getTextPreview()`，解析 JSON。
2. `pdf`、`image`、`video`：调用 `getAuthorizedBlob()`，拿到 Blob 后创建临时 URL。

渲染方式：

| 类型 | 前端元素 |
| --- | --- |
| `image` | `<img>` |
| `video` | `<video controls preload="metadata">` |
| `pdf` | `<iframe>` |
| `markdown` | `ReactMarkdown` |
| `text` | `<pre>` |

关闭预览弹窗时，会调用 `URL.revokeObjectURL()` 释放 Blob URL。

## 4. 前端调用建议

判断是否可预览：

```ts
const previewable = file.preview_kind !== "none" && file.status === "available";
```

媒体类预览：

```ts
const { blob } = await getAuthorizedBlob(token, `/api/v1/admin/files/${file.id}/preview`);
const url = URL.createObjectURL(blob);
```

图片：

```tsx
<img src={url} alt={file.original_name} />
```

视频：

```tsx
<video src={url} controls preload="metadata" />
```

关闭时释放：

```ts
URL.revokeObjectURL(url);
```

## 5. 本机测试命令

运行后端测试：

```bash
go test ./...
```

运行前端构建：

```bash
npm run build
```

本机启动后端：

```bash
DATA_DIR=./data ADDR=:19090 go run .
```

登录管理员：

```bash
LOGIN_RESP=$(curl -sS http://127.0.0.1:19090/api/v1/admin/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"password123"}')

TOKEN=$(echo "$LOGIN_RESP" | sed -n 's/.*"access_token":"\([^"]*\)".*/\1/p')
```

上传图片：

```bash
curl -sS http://127.0.0.1:19090/api/v1/files \
  -H "Authorization: Bearer $TOKEN" \
  -F 'file=@/path/to/example.png'
```

上传视频：

```bash
curl -sS http://127.0.0.1:19090/api/v1/files \
  -H "Authorization: Bearer $TOKEN" \
  -F 'file=@/path/to/example.mp4'
```

筛选图片：

```bash
curl -sS 'http://127.0.0.1:19090/api/v1/admin/files?preview_kind=image' \
  -H "Authorization: Bearer $TOKEN"
```

筛选视频：

```bash
curl -sS 'http://127.0.0.1:19090/api/v1/admin/files?preview_kind=video' \
  -H "Authorization: Bearer $TOKEN"
```

预览文件：

```bash
FILE_ID=1
curl -I http://127.0.0.1:19090/api/v1/admin/files/$FILE_ID/preview \
  -H "Authorization: Bearer $TOKEN"
```

预期可以看到：

```text
Content-Type: image/png
```

或：

```text
Content-Type: video/mp4
```

## 6. 部署注意事项

1. 更新后首次启动会自动迁移 SQLite 表结构。
2. 部署前建议备份服务器上的 `data/` 目录。
3. 不要上传本地 `data/` 覆盖服务器数据。
4. 视频文件仍受 `MAX_UPLOAD_BYTES` 限制，默认 50 MiB。
5. 如果需要上传更大的视频，可调整环境变量 `MAX_UPLOAD_BYTES`，并同步调整反向代理请求体大小限制。
