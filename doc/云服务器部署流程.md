# 后端云服务器部署流程

版本：v0.1  
适用范围：将当前 Go + SQLite 文件传输后端部署到云服务器  
推荐方式：Docker Compose  
外部访问端口：`19090`

## 1. 部署目标

后端服务部署后提供：

```text
http://服务器IP:19090
```

健康检查地址：

```text
http://服务器IP:19090/healthz
```

容器内部仍监听 `8080`，服务器外部端口映射为 `19090`：

```yaml
ports:
  - "19090:8080"
```

## 2. 服务器前置条件

云服务器需要安装：

1. Docker
2. Docker Compose 插件

检查命令：

```bash
docker --version
docker compose version
```

如果命令不存在，需要先按服务器系统安装 Docker。常见 Linux 服务器可以参考：

```bash
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker
```

如果当前用户没有 Docker 权限，可以临时使用 `sudo docker ...`，或将用户加入 `docker` 组。

## 3. 需要上传哪些文件

如果选择在服务器上构建 Docker 镜像，至少需要上传这些文件：

```text
main.go
go.mod
go.sum
Dockerfile
docker-compose.yml
.dockerignore
```

建议同时上传：

```text
README.md
doc/
```

不需要上传：

```text
data/
tmp/
trash/
*.db
*.db-*
file-transfer-backend
dist/
node_modules/
```

说明：

1. `data/` 是运行后产生的数据目录，应该在服务器上持久化保存，不应该用本地测试数据覆盖。
2. `dist/`、`node_modules/` 属于前端或构建产物，不是后端 Docker 构建必需文件。
3. `.dockerignore` 会避免无关文件进入 Docker 构建上下文。

## 4. 推荐目录结构

服务器上建议放到：

```text
/opt/file-transfer-backend/
  main.go
  go.mod
  go.sum
  Dockerfile
  docker-compose.yml
  .dockerignore
  README.md
  doc/
  data/
```

创建目录：

```bash
sudo mkdir -p /opt/file-transfer-backend
sudo chown -R $USER:$USER /opt/file-transfer-backend
```

## 5. 上传文件到服务器

在本机项目目录执行：

```bash
rsync -av \
  --exclude 'data/' \
  --exclude 'tmp/' \
  --exclude 'trash/' \
  --exclude '*.db' \
  --exclude '*.db-*' \
  --exclude 'node_modules/' \
  --exclude 'dist/' \
  --exclude 'file-transfer-backend' \
  ./ 用户名@服务器IP:/opt/file-transfer-backend/
```

示例：

```bash
rsync -av \
  --exclude 'data/' \
  --exclude 'tmp/' \
  --exclude 'trash/' \
  --exclude '*.db' \
  --exclude '*.db-*' \
  --exclude 'node_modules/' \
  --exclude 'dist/' \
  --exclude 'file-transfer-backend' \
  ./ root@1.2.3.4:/opt/file-transfer-backend/
```

如果没有 `rsync`，也可以用 `scp`：

```bash
scp main.go go.mod go.sum Dockerfile docker-compose.yml .dockerignore README.md 用户名@服务器IP:/opt/file-transfer-backend/
scp -r doc 用户名@服务器IP:/opt/file-transfer-backend/
```

## 6. 在服务器上构建并启动

进入目录：

```bash
cd /opt/file-transfer-backend
```

构建并启动：

```bash
docker compose up -d --build
```

查看容器状态：

```bash
docker compose ps
```

查看日志：

```bash
docker compose logs -f
```

验证健康检查：

```bash
curl http://127.0.0.1:19090/healthz
```

预期响应：

```json
{"status":"ok"}
```

从本机访问：

```bash
curl http://服务器IP:19090/healthz
```

## 7. 云服务器安全组和防火墙

需要在云服务器控制台安全组中放行 TCP `19090`。

如果服务器系统启用了防火墙，也需要放行：

Ubuntu 使用 `ufw`：

```bash
sudo ufw allow 19090/tcp
sudo ufw status
```

CentOS / Rocky Linux 使用 `firewalld`：

```bash
sudo firewall-cmd --add-port=19090/tcp --permanent
sudo firewall-cmd --reload
sudo firewall-cmd --list-ports
```

## 8. 首次部署后的管理员账号

默认管理员：

```text
username: admin
password: password123
```

首次部署后，应立即修改密码。

登录获取 Token：

```bash
LOGIN_RESP=$(curl -sS http://127.0.0.1:19090/api/v1/admin/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"password123"}')

echo "$LOGIN_RESP"
```

如果服务器安装了 `jq`：

```bash
TOKEN=$(echo "$LOGIN_RESP" | jq -r '.data.access_token')
```

没有 `jq` 可以用：

```bash
TOKEN=$(echo "$LOGIN_RESP" | sed -n 's/.*"access_token":"\([^"]*\)".*/\1/p')
```

修改密码：

```bash
curl -i -X PATCH http://127.0.0.1:19090/api/v1/admin/password \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"old_password":"password123","new_password":"请替换成你的新密码"}'
```

成功时返回：

```text
HTTP/1.1 204 No Content
```

## 9. 上传和下载验证

上传测试文件：

```bash
echo 'hello server' > /tmp/hello.txt

UPLOAD_RESP=$(curl -sS http://127.0.0.1:19090/api/v1/files \
  -F 'file=@/tmp/hello.txt')

echo "$UPLOAD_RESP"
```

提取文件码：

```bash
CODE=$(echo "$UPLOAD_RESP" | sed -n 's/.*"code":"\([^"]*\)".*/\1/p')
echo "$CODE"
```

下载验证：

```bash
curl -sS http://127.0.0.1:19090/api/v1/files/$CODE/download
```

管理员查看文件列表：

```bash
curl -sS http://127.0.0.1:19090/api/v1/admin/files \
  -H "Authorization: Bearer $TOKEN"
```

rsync -av \

  --exclude 'trash/' \
  --exclude '*.db' \

```bash

```

## 12. 数据持久化和备份

当前 `docker-compose.yml` 将服务器当前目录下的 `./data` 挂载到容器 `/data`：

```yaml
volumes:
  - ./data:/data
```

`data/` 中包含：

```text
data/
  app.db
  uploads/
  tmp/
  trash/
```

备份建议：

```bash
cd /opt/file-transfer-backend
tar -czf file-transfer-backend-data-$(date +%F).tar.gz data
```

恢复时，先停止服务：

```bash
docker compose down
```

再恢复 `data/` 目录，然后启动：

```bash
docker compose up -d
```

注意：不要在服务运行中随意覆盖 `data/app.db`。

## 13. 常见问题

### 13.1 访问 `服务器IP:19090` 不通

检查顺序：

```bash
docker compose ps
docker compose logs --tail=100
curl http://127.0.0.1:19090/healthz
```

如果服务器本机能访问，但外部不能访问，通常是云安全组或系统防火墙没有放行 `19090`。

### 13.2 端口冲突

检查端口：

```bash
sudo lsof -nP -iTCP:19090 -sTCP:LISTEN
```

如果被占用，修改 `docker-compose.yml`：

```yaml
ports:
  - "新的外部端口:8080"
```

然后重启：

```bash
docker compose up -d
```

### 13.3 忘记管理员密码

当前实现会把管理员密码哈希保存在 `data/app.db` 中。已有数据库不会因为重启而重置密码。

如果忘记密码，需要停服后处理数据库，或备份并重建 `data/`。重建 `data/` 会丢失已有文件和元数据，因此生产环境不要直接删除。

### 13.4 上传文件失败

重点检查：

1. 文件是否超过 `MAX_UPLOAD_BYTES`。
2. `data/` 目录是否可写。
3. 容器日志中是否有写入错误。

查看日志：

```bash
docker compose logs --tail=100
```

## 14. 生产部署建议

1. 使用 HTTPS 或在可信内网中访问。
2. 部署后立即修改默认管理员密码。
3. 定期备份 `/opt/file-transfer-backend/data`。
4. 不要把 `data/` 覆盖上传。
5. 云服务器安全组只开放必要端口。
6. 如果前端和后端部署在同一域名下，建议由 Nginx 或 Caddy 反向代理到 `127.0.0.1:19090`。
