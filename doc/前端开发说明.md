# 文件传输后端前端开发说明

版本：v0.1  
适用对象：前端开发、联调、接口测试  
后端基础地址：本机默认 `http://127.0.0.1:8080`

## 1. 项目简介

这是一个轻量文件传输后端，用于小文件上传、凭 6 位文件码下载，以及管理员文件管理。

核心能力：

1. 匿名用户可以上传文件，上传成功后获得唯一文件码。
2. 匿名用户只能凭文件码下载文件，不能查看文件列表。
3. 管理员可以登录、查看文件列表、下载、预览文本/Markdown/PDF/图片/视频、删除、批量删除文件。
4. 管理员可以查看上传、下载、删除事件，并删除事件记录。
5. 文件内容存储在服务器文件系统，元数据存储在 SQLite。
6. 上传和下载均为流式处理，适合低内存服务器部署。

默认管理员账号：

```text
username: admin
password: password123
```

部署或本机首次启动后，建议先通过改密接口修改默认密码。

## 3. 通用 API 约定

API 基础路径：

```text
/api/v1
```

JSON 成功响应统一格式：

```json
{
  "request_id": "req_ABC123",
  "data": {}
}
```

JSON 错误响应统一格式：

```json
{
  "request_id": "req_ABC123",
  "error": {
    "code": "INVALID_ARGUMENT",
    "message": "file is required",
    "details": {
      "field": "file"
    }
  }
}
```

前端可选传入请求 ID：

```http
X-Request-ID: req_frontend_001
```

如果未传，后端会自动生成。

常见错误码：

| HTTP 状态 | 错误码 | 说明 |
| --- | --- | --- |
| 400 | `INVALID_ARGUMENT` | 参数错误、请求体错误 |
| 401 | `UNAUTHORIZED` | 管理员认证失败 |
| 404 | `NOT_FOUND` | 资源不存在；公共下载会统一返回该错误 |
| 413 | `PAYLOAD_TOO_LARGE` | 上传文件过大 |
| 415 | `UNSUPPORTED_MEDIA_TYPE` | Content-Type 不支持 |
| 422 | `UNSUPPORTED_PREVIEW` | 不支持预览 |
| 500 | `INTERNAL` | 服务端内部错误 |

## 4. 管理员认证

管理员登录成功后，前端需要保存 `access_token`，后续管理员接口带上：

```http
Authorization: Bearer <access_token>
```

Token 过期、缺失或已吊销时，接口返回 `401 UNAUTHORIZED`。

### 4.1 登录

```http
POST /api/v1/admin/login
Content-Type: application/json
```

请求：

```json
{
  "username": "admin",
  "password": "password123"
}
```

curl：

```bash
curl -sS http://127.0.0.1:8080/api/v1/admin/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"password123"}'
```

响应：

```json
{
  "request_id": "req_ABC123",
  "data": {
    "access_token": "adm_sess_xxx",
    "token_type": "Bearer",
    "expires_at": "2026-05-09T10:20:30.123Z",
    "admin": {
      "id": 1,
      "username": "admin",
      "password_changed_at": "2026-05-08T10:20:30.123Z"
    }
  }
}
```

### 4.2 获取当前管理员

```http
GET /api/v1/admin/me
Authorization: Bearer <access_token>
```

curl：

```bash
TOKEN="<从登录响应中复制 access_token>"
curl -sS http://127.0.0.1:8080/api/v1/admin/me \
  -H "Authorization: Bearer $TOKEN"
```

### 4.3 修改密码

```http
PATCH /api/v1/admin/password
Authorization: Bearer <access_token>
Content-Type: application/json
```

请求：

```json
{
  "old_password": "password123",
  "new_password": "new-password-456"
}
```

成功响应：`204 No Content`

说明：

1. 新密码长度必须为 8 到 128 个字符。
2. 修改成功后，会吊销其他管理员会话。
3. 当前会话仍可继续使用。

### 4.4 退出登录

```http
POST /api/v1/admin/logout
Authorization: Bearer <access_token>
```

成功响应：`204 No Content`

## 5. 公共文件接口

### 5.1 上传文件

```http
POST /api/v1/files
Content-Type: multipart/form-data
```

表单字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `file` | 是 | 要上传的文件，每次只支持 1 个 |

匿名上传 curl：

```bash
echo 'hello frontend' > /tmp/hello.txt
curl -sS http://127.0.0.1:8080/api/v1/files \
  -F 'file=@/tmp/hello.txt'
```

管理员上传 curl：

```bash
curl -sS http://127.0.0.1:8080/api/v1/files \
  -H "Authorization: Bearer $TOKEN" \
  -F 'file=@/tmp/hello.txt'
```

响应：

```json
{
  "request_id": "req_ABC123",
  "data": {
    "code": "AB3DE9",
    "original_name": "hello.txt",
    "size_bytes": 14,
    "mime_type": "text/plain",
    "sha256": "8f434346648f6b96df89dda901c5176b10a6d83961dd3c1ac88b59b2dc327aa4",
    "preview_kind": "text",
    "uploaded_at": "2026-05-08T10:20:30.123Z",
    "download_url": "/api/v1/files/AB3DE9/download"
  }
}
```

前端注意：

1. 上传时不要手动设置 multipart boundary，让浏览器或请求库自动生成 `Content-Type`。
2. 如果带了有效管理员 Token，后端会把 `uploaded_by_role` 记录为 `admin`。
3. 文件名只用于展示和下载，不会作为服务器存储路径。

### 5.2 凭码下载文件

```http
GET /api/v1/files/{code}/download
HEAD /api/v1/files/{code}/download
```

curl：

```bash
CODE="AB3DE9"
curl -OJ http://127.0.0.1:8080/api/v1/files/$CODE/download
```

说明：

1. `{code}` 是 6 位字母数字码，大小写都可以，后端会转成大写。
2. 成功响应是文件二进制流，不包 JSON。
3. 公共下载失败统一返回 `404 NOT_FOUND`，不会区分文件不存在、已删除或码格式错误。
4. `GET` 会记录下载事件并增加下载次数；`HEAD` 只检查响应头，不增加下载次数。
5. 支持浏览器常用的 Range 下载能力。

### 6.3 管理员下载文件

```http
GET /api/v1/admin/files/{file_id}/download
HEAD /api/v1/admin/files/{file_id}/download
```

curl：

```bash
curl -OJ http://127.0.0.1:8080/api/v1/admin/files/$FILE_ID/download \
  -H "Authorization: Bearer $TOKEN"
```

说明：响应为文件流，不包 JSON。

### 6.4 预览文件

```http
GET /api/v1/admin/files/{file_id}/preview
```

不同文件类型响应不同：

| `preview_kind` | 响应 |
| --- | --- |
| `text` | JSON，返回文本内容 |
| `markdown` | JSON，返回 Markdown 原文 |
| `pdf` | `application/pdf` 内联文件流 |
| `image` | 图片内联文件流，例如 `image/png`、`image/jpeg` |
| `video` | 视频内联文件流，例如 `video/mp4`、`video/webm` |
| `none` | `422 UNSUPPORTED_PREVIEW` |

文本和 Markdown 响应：

```json
{
  "request_id": "req_ABC123",
  "data": {
    "kind": "text",
    "encoding": "utf-8",
    "content": "hello\n",
    "truncated": false,
    "bytes_read": 6
  }
}
```

curl：

```bash
curl -sS http://127.0.0.1:8080/api/v1/admin/files/$FILE_ID/preview \
  -H "Authorization: Bearer $TOKEN"
```

前端注意：

1. Markdown 预览返回原文，前端渲染时必须做 XSS 防护。
2. 文本预览只读取前 `TEXT_PREVIEW_BYTES` 字节，可能返回 `truncated: true`。
3. 当前实现要求文本内容为 UTF-8。
4. PDF、图片、视频预览直接返回二进制文件流，不包 JSON。
5. 视频预览支持浏览器 Range 请求，前端可直接放入 `<video controls>`。

### 6.5 删除单个文件

```http
DELETE /api/v1/admin/files/{file_id}
```

成功响应：`204 No Content`

curl：

```bash
curl -i -X DELETE http://127.0.0.1:8080/api/v1/admin/files/$FILE_ID \
  -H "Authorization: Bearer $TOKEN"
```

说明：

1. 删除会移除服务器上的实际文件。
2. 元数据保留，状态改为 `deleted`。
3. 已删除文件重复删除仍返回 `204`。
4. 文件记录不存在返回 `404`。

### 6.6 批量删除文件


请求：


curl：

```bash

  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"file_ids":[1,2,3]}'
```

响应：

```json
{
  "request_id": "req_ABC123",
  "data": {
    "items": [
      {
        "id": 1,
        "code": "AB3DE9",
        "status": "deleted"
      },
      {
```

1. `file_ids` 必须是非空数组。
2. 单次最多 100 个 ID。
3. 后端会自动去重。

## 8. 前端推荐联调流程

### 8.1 最小匿名流程

1. 调用 `POST /api/v1/files` 上传文件。
2. 从响应中读取 `data.code`。
3. 使用 `GET /api/v1/files/{code}/download` 下载文件。
4. 对不存在或已删除的文件码，统一展示“文件不存在或已失效”。

### 8.2 管理员流程

1. 调用 `POST /api/v1/admin/login` 获取 Token。
2. 调用 `GET /api/v1/admin/files` 渲染管理列表。
3. 根据 `preview_kind` 决定是否显示预览按钮。
4. 下载使用 `/api/v1/admin/files/{file_id}/download`。
5. 删除文件后刷新列表或本地将该项移除。
6. 调用 `GET /api/v1/admin/events` 展示事件明细。

### 8.3 浏览器 fetch 示例

上传：

```js
async function uploadFile(file, token) {
  const form = new FormData();
  form.append("file", file);

  const headers = {};
  if (token) headers.Authorization = `Bearer ${token}`;

  const res = await fetch("/api/v1/files", {
    method: "POST",
    headers,
    body: form,
  });

  const payload = await res.json();
  if (!res.ok) throw payload.error;
  return payload.data;
}
```

管理员登录：

```js
async function login(password) {
  const res = await fetch("/api/v1/admin/login", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ username: "admin", password }),
  });

  const payload = await res.json();
  if (!res.ok) throw payload.error;
  return payload.data.access_token;
}
```

管理员列表：

```js
async function listFiles(token, page = 1) {
  const res = await fetch(`/api/v1/admin/files?page=${page}&page_size=20`, {
    headers: { Authorization: `Bearer ${token}` },
  });

  const payload = await res.json();
  if (!res.ok) throw payload.error;
  return payload.data;
}
```

下载：

```js
function publicDownloadUrl(code) {
  return `/api/v1/files/${encodeURIComponent(code)}/download`;
}

function adminDownloadUrl(fileId) {
  return `/api/v1/admin/files/${fileId}/download`;
}
```

## 9. 一组可复制的本机联调命令

启动：

```bash
DATA_DIR=./data ADDR=:8080 go run .
```

另开终端执行：

```bash
curl -sS http://127.0.0.1:8080/healthz

echo 'hello frontend' > /tmp/hello.txt

UPLOAD_RESP=$(curl -sS http://127.0.0.1:8080/api/v1/files \
  -F 'file=@/tmp/hello.txt')
echo "$UPLOAD_RESP"

CODE=$(echo "$UPLOAD_RESP" | sed -n 's/.*"code":"\([^"]*\)".*/\1/p')
curl -sS "http://127.0.0.1:8080/api/v1/files/$CODE/download"

LOGIN_RESP=$(curl -sS http://127.0.0.1:8080/api/v1/admin/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"password123"}')
echo "$LOGIN_RESP"

TOKEN=$(echo "$LOGIN_RESP" | sed -n 's/.*"access_token":"\([^"]*\)".*/\1/p')

curl -sS http://127.0.0.1:8080/api/v1/admin/me \
  -H "Authorization: Bearer $TOKEN"

curl -sS http://127.0.0.1:8080/api/v1/admin/files \
  -H "Authorization: Bearer $TOKEN"

curl -sS http://127.0.0.1:8080/api/v1/admin/events \
  -H "Authorization: Bearer $TOKEN"
```

如果本机安装了 `jq`，解析 Token 和文件码更稳定：

```bash
CODE=$(echo "$UPLOAD_RESP" | jq -r '.data.code')
TOKEN=$(echo "$LOGIN_RESP" | jq -r '.data.access_token')
```

## 10. 前端实现注意事项

1. 管理员 Token 建议存放在内存或安全存储中，退出登录后清理。
2. 遇到 `401` 时跳转登录或提示会话过期。
3. 公共下载的 `404` 不应展示过细原因，统一提示文件不存在或已失效。
4. 上传失败时优先根据 `error.code` 分支处理，比如文件过大对应 `PAYLOAD_TOO_LARGE`。
5. 文本和 Markdown 预览由 JSON 返回；PDF、图片、视频预览直接使用文件流 URL。
6. Markdown 渲染必须做 XSS 防护。
7. 文件列表分页使用 `page`、`page_size` 和 `has_more` 控制加载更多。
8. 删除接口成功可能是 `204` 空响应，前端不要强行解析 JSON。
